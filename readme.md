## 1. Реализация АТД LazySequence<T>

Создан шаблонный класс LazySequence<T>, который представляет последовательность, элементы которой вычисляются по требованию (лениво) и кэшируются (мемоизация).

Класс использует внутренний Generator<T> для порождения новых элементов.

Управление длиной выполнено через специальный тип Cardinal, способный представлять как конечное число элементов, так и бесконечность.

Реализованы методы:

GetFirst(), GetLast(), Get(index)

GetSubsequence(start, end)

GetLength() (возвращает Cardinal)

GetMaterializedCount()

Append(), Prepend(), InsertAt(), Concat()

Map(), Where(), Reduce(), Zip()

GetEnumerator() для перебора

Ленивые последовательности позволяют работать с потенциально бесконечными данными (например, последовательность Фибоначчи), не выделяя память под все элементы сразу. Также они полезны для обработки очень больших объёмов данных, которые не могут быть целиком загружены в память.

##### Как это работает?
При создании LazySequence передаётся генератор – объект, умеющий по запросу выдавать следующий элемент.

При обращении к элементу по индексу вызывается метод MaterializeUntil(index), который последовательно получает от генератора недостающие элементы и сохраняет их во внутренний массив memoized.

После материализации доступ к элементу происходит за O(1).

Операции вроде Concat создают новый объект LazySequence, комбинирующий генераторы исходных последовательностей без немедленной материализации.

Длина, равная бесконечности, обозначается через Cardinal со специальным значением; методы GetLast() для бесконечной последовательности выбрасывают исключение.

## 2. Реализация АТД ReadOnlyStream<T>
Создан абстрактный базовый класс ReadOnlyStream<T>, определяющий интерфейс для последовательного чтения элементов из некоторого источника.

Объявлены виртуальные методы: Open(), Close(), Read(), IsEndOfStream(), GetPosition().

Предоставлены реализации по умолчанию для Seek(), IsCanSeek(), IsCanGoBack() (выбрасывают исключение).

Реализованы конкретные производные классы:

StringStream – чтение слов из строки (используется в алфавитном указателе).

FileReadOnlyStream – чтение из файла.

Поток является обобщением источника данных. Задача требует уметь обрабатывать данные, поступающие из разных мест (память, файл, сеть), одинаковым образом. Абстракция потока скрывает детали источника.

##### Как это работает
Клиент вызывает Open() для подготовки ресурса, затем в цикле проверяет IsEndOfStream() и читает элементы через Read().

Read() перемещает внутреннюю позицию и возвращает следующий элемент; при достижении конца выбрасывается исключение EndOfStream.

StringStream при создании разбивает строку на слова (по пробелам и знакам препинания).

FileReadOnlyStream работает с библиотекой <fstream>, читая данные по одному слову.


## 3. Тематическая задача 1: Построение алфавитного указателя
Класс AlphabeticalIndex построен на основе ReadOnlyStream<std::string> и BinarySearchTree<std::string, MutableArraySequence<int>*>.

Выполнена очистка слов: приведение к нижнему регистру, удаление неалфавитно-цифровых символов.

Каждому слову сопоставляется список номеров страниц, где оно встречается.

Страница вычисляется как (позиция_в_потоке / words_per_page) + 1, где words_per_page = 50.

Реализован метод BuildFromStream(), который заполняет индекс, и Print(), выводящий указатель в алфавитном порядке.

Алфавитный указатель (конкорданс) – классическая задача обработки текста. В рамках лабораторной работы она демонстрирует совместное использование потока (ReadOnlyStream) и структуры данных (бинарное дерево поиска) для накопления и последующей сортированной выдачи информации.

##### Как это работает
Поток открывается, и из него читаются слова одно за другим.

Для каждого слова вычисляется текущий номер страницы на основе GetPosition().

Слово очищается (функция CleanWord).

В бинарном дереве отыскивается или создаётся узел с ключом-словом, а значением является указатель на список страниц.

Если номер страницы ещё не присутствует в списке, он добавляется в конец.

После исчерпания потока выполняется обход дерева в симметричном порядке (in-order), что автоматически даёт слова в алфавитном порядке; для каждого слова печатается его список страниц.

## 4. Тематическая задача 2: Машина Тьюринга
   Класс TuringMachine симулирует детерминированную машину Тьюринга.

Лента представлена как MutableArraySequence<char> с автоматическим расширением при выходе головки за пределы.

Таблица переходов реализована как std::map<std::string, std::map<char, Transition>>, где Transition содержит символ записи, направление движения и следующее состояние.

Доступны методы: AddTransition(), SetInput(), Step(), Run(), GetTape(), GetState(), GetHeadPosition().

Поддерживается специальный символ _ (пробел) как символ пустой ячейки (blank).

При движении влево за начало ленты выполняется автоматическое добавление пустого символа в начало (сдвиг всех элементов).

Машина Тьюринга – фундаментальная модель вычислений. Её реализация демонстрирует работу с динамическими последовательностями, обработку правил переходов и конечных автоматов – всё это требуется в задании.

##### Как это работает
Лента инициализируется через SetInput(), которая записывает символы входной строки в последовательность.

Головка изначально указывает на первый символ (позиция 0).

Метод Step() выполняет один такт:

Считывается символ под головкой.

По текущему состоянию и считанному символу ищется переход в таблице.

Если переход найден, на ленту записывается новый символ, головка перемещается (влево/вправо/стоит), состояние изменяется.

При выходе за левую границу лента расширяется добавлением пустого символа в начало, а головка смещается на вновь добавленную позицию.

При выходе за правую границу лента расширяется добавлением пустого символа в конец.

Метод возвращает true, если переход был выполнен, и false, если перехода нет (машина остановилась).

Метод Run(maxSteps) вызывает Step() циклически до остановки или до достижения лимита шагов.

Архитектурные решения
Динамическая лента – использование MutableArraySequence позволяет не задавать фиксированную длину заранее. Это упрощает работу с потенциально бесконечной лентой.

Автоматическое дополнение пробелами – при выходе за границы лента расширяется с пустыми символами, что моделирует бесконечную ленту. При расширении слева фактически все элементы сдвигаются – это O(n), но в реальных сценариях сдвиг происходит относительно редко.

Таблица переходов как вложенный ассоциативный массив – обеспечивает быстрый поиск перехода по состоянию и символу. Ключи состояний – строки, что удобно для отладки.

Пошаговое выполнение – метод Step() позволяет пошагово отслеживать работу, что полезно для отладки и демонстрации.
## 5. Тематическая задача 3: Синхронизация данных по событиям
Класс SyncEventGenerator<T> (в демонстрации T = double) синхронизирует несколько потоков временных данных.

Каждый входной поток представлен последовательностью пар (timestamp, value).

Параметр τ (tau) задаёт интервал, внутри которого события считаются относящимися к одному моменту.

Для каждого окна вычисляется оценка времени события как среднее арифметическое меток времени в окне минус τ/2.

Выходной элемент содержит оценённое время и вектор из std::optional<T> (значения отсутствующих потоков – nullopt).

Класс реализует интерфейс Generator<Event<T>>, что позволяет использовать его внутри LazySequence.

В реальных системах сбора данных показания с разных датчиков поступают не строго одновременно и могут пропадать. Задача требует восстановить единую временную сетку событий, оценив истинное время и связав с ним все имеющиеся показания. Это моделирует, например, работу триггера в измерительной системе.

##### Как это работает
В конструктор передаются векторы из указателей на Sequence<Pair<double, T>> – исходные данные для каждого потока, а также значение τ.

Для каждого потока создаётся перечислитель (IEnumerator<Pair<double, T>>) и извлекается первый элемент.

Метод HasNext() возвращает true, пока хотя бы один перечислитель не исчерпан.

Метод GetNext():

Находит наименьшую метку времени t_min среди текущих элементов всех потоков, у которых есть данные.

Собирает значения из всех потоков, чья текущая метка времени <= t_min + τ.

Для каждого такого потока перечислитель переводится на следующий элемент.

Вычисляется оценённое время: (sum_of_timestamps) / count - τ/2.

Возвращается объект Event<T>, содержащий оценённое время и вектор значений (пропуски обозначены nullopt).

##### Архитектурные решения
Использование перечислителей (IEnumerator) – каждый входной поток оборачивается в единый интерфейс, что абстрагирует источник данных (это может быть массив, файл, ленивая последовательность).

Онлайновый алгоритм – в каждый момент времени хранится только текущий элемент каждого потока, никакого буфера прошедших событий. Это позволяет обрабатывать бесконечные последовательности с минимальным потреблением памяти.

Явное обозначение пропусков – std::optional<T> делает намерение «нет данных» явным, в отличие от использования специальных значений вроде NaN.

Соответствие формуле из методички – оценка времени точно следует формуле (∑t_i)/n - τ/2, что подтверждает следование постановке задачи.

Возможность использования как генератора для LazySequence – класс наследует от Generator<Event<T>>, поэтому может быть передан в конструктор ленивой последовательности для порождения синхронизированных событий «на лету».
